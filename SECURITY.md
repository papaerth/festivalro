# 🔒 축제로 정기 보안 점검 (한 달에 한 번)

> 코딩을 몰라도 됩니다. 아래 **"복붙용 지시문"** 전체를 그대로 복사해서
> Claude Code(클로드 코드)에게 붙여넣기만 하면 됩니다.
> 클로드가 검사 → 안전한 업데이트 → 화면 확인 → 배포까지 순서대로 해줍니다.

---

## 📅 언제 하나요?

- **한 달에 한 번** (예: 매월 1일). 달력에 반복 일정으로 넣어두세요.
- 뉴스에서 "심각한 보안 취약점" 소식을 들었을 때 추가로 한 번.

---

## 📋 복붙용 지시문 (이 아래 박스 전체를 복사)

```
정기 보안 점검을 시작해줘. SECURITY.md의 절차를 따라줘. 순서는 이렇게:

1. [취약점 검사] npm audit 를 실행해서 현재 취약점을 심각도별로 보고해줘.

2. [안전한 범위 업데이트]
   - high 또는 critical 취약점이 있으면: 같은 메이저 버전 안에서 고칠 수 있는
     안전한 방법(패치/마이너 또는 overrides)으로 수정해줘.
   - 고치려면 메이저 버전 업(예: next 15 → 16)이나 파괴적 변경이 필요한 경우에는
     '절대 바로 하지 말고' 먼저 나에게 위험도와 선택지를 설명하고 물어봐줘.
   - moderate 이하만 있고 고치는 방법이 파괴적 변경뿐이면, 위험 설명만 하고
     업데이트는 보류해줘.

3. [화면 확인] 업데이트를 했다면 npm run build 로 빌드가 성공하는지 확인하고,
   주요 화면(홈 검색, 카드뉴스, 지도, 축제 상세페이지, 영상 쇼츠)이
   정상인지 스크린샷 등으로 점검해줘. 하나라도 깨지면 업데이트를 되돌려줘.

4. [배포] 위 3번까지 모두 정상이면 git 커밋 후 push 하고,
   라이브 사이트(chukjero.com)에서 정상 동작을 확인해줘.

마지막에 3줄 요약을 줘:
- 오늘 조치한 것
- 내가 직접 해야 할 것(있으면)
- 다음 정기 점검 예정일(한 달 뒤 날짜)

하지 말 것: 유료 보안 서비스 도입 금지, 내 확인 없이 대규모(메이저) 업그레이드 금지,
기능을 깨뜨리는 변경 금지.
```

## 🙋 내가 직접(사람이) 챙길 것 — 클로드가 대신 못 하는 것

한 달에 한 번, 아래도 같이 확인하면 좋습니다.

- [ ] **GitHub 2단계 인증(2FA)** 켜져 있는지
      → github.com → Settings → Password and authentication
- [ ] **Vercel 2단계 인증(2FA)** 켜져 있는지
      → vercel.com → Account Settings → Authentication
- [ ] **API 키 사용량 이상 없는지** (누가 몰래 쓰고 있지 않은지)
      → Google Cloud Console(YouTube), 네이버 개발자센터, data.go.kr 에서 사용량 확인
- [ ] **YouTube 키 제한** 걸려 있는지 (API 제한 = YouTube Data API v3)
      → console.cloud.google.com/apis/credentials
- [ ] 이메일로 온 **GitHub/Vercel 보안 알림(Dependabot 등)** 이 있으면 클로드에게 전달

## 🧩 지금 우리 사이트에 적용돼 있는 보안 (참고용)

- ✅ API 키는 전부 서버에서만 사용 (`.env.local`, 브라우저로 안 나감)
      - 키 읽는 파일엔 `server-only` 가드 → 실수로 클라이언트에 넣으면 빌드 실패
- ✅ 모든 외부 API 호출은 서버(`/api/...`)를 거침 (키 노출 없음)
- ✅ API 남용 방지: IP별 호출 횟수 제한 (`src/lib/rateLimit.js`의 숫자로 조정 가능)
- ✅ 응답 캐싱으로 외부 API 한도 절약 (blog/videos/overview/weather 등)
- ✅ 표준 보안 헤더 (`next.config.mjs`): 클릭재킹·MIME 스니핑 방지, HTTPS 강제 등
- ✅ 외부 링크에 `rel="noopener noreferrer"` (탭 탈취 방지)
- ✅ 이미지 프록시는 허용된 호스트(naver/pstatic)만 (오남용 방지)
- ✅ React 자동 이스케이프로 XSS 방지 (위험한 `dangerouslySetInnerHTML` 미사용)
