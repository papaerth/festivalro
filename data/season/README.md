# 개화·단풍 시즌 정보 갱신 안내 (기획자용)

이 폴더의 두 파일이 사이트의 "🌸 개화 예상 / 지금 절정", "🍁 단풍 절정" 배지와
메인 상단 배너의 근거 데이터입니다. **공식 예측 API가 없어** 운영자가 **1년에 2번**
공식 발표를 보고 직접 갱신하는 방식입니다. (코드 수정 아님 — 날짜 숫자만 바꾸면 됩니다.)

## 언제 갱신하나
- **봄 (매년 2월 말):** 산림청이 "봄철 꽃나무(벚꽃) 개화 예측지도"를 발표하면 → `bloom.json` 갱신
- **가을 (매년 10월 초):** 산림청이 "산림단풍 예측지도"를 발표하면 → `foliage.json` 갱신

출처(검색어): 산림청 보도자료 "봄꽃 개화 예측" / "산림단풍 예측지도", 기상청 봄꽃개화현황.
※ 뉴스·블로그의 지역별 표를 참고해도 됩니다. 크롤링(자동수집)은 하지 않습니다.

## 어떻게 갱신하나
날짜(YYYY-MM-DD)만 바꾸면 됩니다. 권역 7개: seoul(서울)·gyeonggi(경기·인천)·
gangwon(강원)·chungcheong(충청·대전·세종)·jeolla(전라·광주)·gyeongsang(경상·부산·대구·울산)·jeju(제주)

### bloom.json (봄 벚꽃)
- `bloom`: 개화 예상일, `full`: 만개 예상일
- `year`: 해당 연도 숫자로 갱신

### foliage.json (가을 단풍)
- `peakStart` ~ `peakEnd`: 단풍 절정 예상 기간
- `year`: 해당 연도 숫자로 갱신

## 판정 규칙 (자동)
축제 지역의 예상일과 오늘 날짜를 비교해 배지가 자동으로 바뀝니다.
- 봄: 개화 전(개화 예상일 30일 전부터) → 절정(개화~만개+6일) → 끝물(만개+13일까지) → 이후 숨김
- 가을: 단풍 전(30일 전부터) → 절정(기간 내) → 끝물(종료+7일까지) → 이후 숨김
- 시즌이 아니면 배지·배너 **자동 숨김**. 연도는 무시하고 월·일만 비교하므로,
  갱신을 한 해 걸러도 대략적으로는 계속 동작합니다(그래도 매년 갱신 권장).

배지가 붙는 축제는 제목에 꽃/단풍 관련 단어(벚꽃·매화·유채·진달래…/단풍·억새·은행…)가
있는 축제입니다. 대상 단어는 `src/lib/season.js`의 BLOOM_RE·FOLIAGE_RE에서 조정할 수 있습니다.
