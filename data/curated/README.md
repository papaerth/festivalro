# 축제 큐레이션 정보 추가하기 (코딩 몰라도 OK)

이 폴더(`data/curated/`)에 **축제별 JSON 파일**을 하나 넣으면, 그 축제 상세페이지에
**타임테이블·라인업·셔틀버스·주차·먹거리·꿀팁·현장시설·외국인 안내** 섹션이 자동으로 나타납니다.

- 파일이 **있는 축제만** 해당 섹션이 보입니다. (없으면 아무것도 안 보임 — 빈 섹션 없음)
- 모든 항목은 **선택사항**이에요. 넣고 싶은 것만 넣으면 됩니다.
- 내용은 **한국어로만** 써도 됩니다. 외국어 사용자에게는 섹션 **제목만 번역**되고,
  내용은 한국어 원문 옆에 `(Korean)` 표시가 붙습니다. (자동 번역은 다음 단계에서 추가 예정)

---

## 1. 새 축제 추가하는 법 (3단계)

### ① 콘텐츠ID(파일 이름) 찾기
1. 축제로 사이트에서 그 축제의 **상세페이지**를 엽니다.
2. 브라우저 **주소창(URL)** 을 보세요. 예: `https://festivalro.vercel.app/festival/142064`
3. 맨 뒤의 숫자 **`142064`** 가 이 축제의 **콘텐츠ID**입니다. → 파일 이름은 **`142064.json`**
   - (영어 페이지면 `/en/festival/142064` 처럼 나오는데, 숫자는 똑같습니다.)
   - 숫자가 아니라 `s...`로 시작하면 표준데이터 축제인데, 그 값이 그대로 파일 이름이 됩니다.

### ② 파일 만들기
1. 이 폴더에 있는 **`506534.json`** 파일을 복사합니다.
2. 파일 이름을 위에서 찾은 **콘텐츠ID**로 바꿉니다. (예: `142064.json`)
3. 안의 내용을 그 축제에 맞게 고쳐 씁니다. (아래 2번 설명 참고)
4. 필요 없는 항목은 **통째로 지워도** 됩니다.

### ③ 반영하기
파일을 저장하고 **git push**(또는 담당자에게 전달)하면, 잠시 뒤 사이트에 자동 반영됩니다.

> 💡 **주의**: JSON은 **쉼표·따옴표·중괄호**가 정확해야 해요. 저장 전에
> [jsonlint.com](https://jsonlint.com) 같은 곳에 붙여넣어 "Valid" 인지 확인하면 안전합니다.

## 2. 넣을 수 있는 항목 (모두 선택사항)

| 항목 | 화면 섹션 | 형식 | 예시 |
|---|---|---|---|
| `timetable` | 🗓️ 타임테이블 (날짜별 탭) | 목록 `{date, time, title}` | 아래 참고 |
| `lineup` | 🎤 라인업 | 목록 `{name, date, time}` | 초대가수/공연팀 |
| `shuttle` | 🚌 셔틀버스 | 글(텍스트) | 셔틀 운행 안내 |
| `parking` | 🅿️ 주차 안내 | 글(텍스트) | 주차장 위치·요금 |
| `food` | 🍢 먹거리 | 목록 `{name, desc}` | 대표 먹거리 |
| `tips` | 💡 방문 꿀팁 | 글 목록 `[ "...", "..." ]` | 꿀팁 여러 개 |
| `facilities` | 🚻 현장 시설 | 글(텍스트) | 화장실·수유실·의무실 등 |
| `foreigner` | 🌏 외국인 방문객 안내 | `{interpretation, halal, payment}` | 통역·할랄/채식·결제 |
| `videos` | 🎬 영상 미리보기(맨 위 임베드) | 목록 `{platform, url}` | 인스타/틱톡/유튜브 게시물 |

### 형식 예시

```json
{
  "timetable": [
    { "date": "2026-07-17", "time": "19:30", "title": "개막 축하공연" },
    { "date": "2026-07-18", "time": "20:00", "title": "EDM 나이트 파티" }
  ],
  "lineup": [
    { "name": "싸이 (PSY)", "date": "2026-07-17", "time": "19:30" }
  ],
  "shuttle": "대천역 ↔ 축제장 무료 셔틀 (09:00~22:00, 20분 간격)",
  "parking": "임시주차장 5곳, 1일 5,000원. 주말은 대중교통 권장.",
  "food": [
    { "name": "대천 조개구이", "desc": "숯불에 구운 서해안 조개" }
  ],
  "tips": [
    "수건과 여벌옷을 챙기세요.",
    "오전이나 저녁이 덜 붐빕니다."
  ],
  "facilities": "화장실 8곳 · 유료 샤워장 · 수유실 · 의무실 운영.",
  "foreigner": {
    "interpretation": "종합안내소에서 영어·중국어·일본어 통역 지원(10:00~18:00)",
    "halal": "채식(비건) 가능 매대가 푸드존 B구역에 있습니다.",
    "payment": "대부분 카드 결제 가능. 일부 노점은 현금만."
  }
}
```

- **날짜 형식**은 `YYYY-MM-DD`(예 `2026-07-17`)로 써주세요. 타임테이블이 날짜별 탭으로 자동 정리됩니다.
- **`timetable`, `lineup`, `food`, `tips`** 는 대괄호 `[ ]` 안에 여러 개를 쉼표로 나열합니다.
- 항목을 안 쓰려면 그 줄을 지우면 됩니다. (단, 바로 앞 항목의 끝 쉼표도 정리해 주세요.)

## 2-1. 🎬 영상(인스타·틱톡·유튜브) 추가하는 법

축제 상세페이지의 **"🎬 영상으로 미리 보는 축제"** 섹션에는 유튜브 쇼츠가 **자동으로** 나옵니다.
여기에 더해, **직접 고른 특정 게시물**을 그 섹션 맨 위에 공식 플레이어로 띄우고 싶으면
`videos` 항목을 넣으면 됩니다. (안 넣어도 유튜브 쇼츠는 자동으로 나와요 — 이건 선택사항이에요.)

**하는 법: 게시물 주소(URL)만 복사해서 붙여넣기.**

1. 인스타그램/틱톡/유튜브에서 그 게시물을 엽니다.
2. 게시물의 **주소(URL)** 를 복사합니다.
   - 인스타: 게시물 오른쪽 위 `···` → **"링크 복사"** (예: `https://www.instagram.com/reel/XXXXXXXXXXX/`)
   - 틱톡: **"공유" → "링크 복사"** (예: `https://www.tiktok.com/@사용자/video/1234567890123456789`)
     - ⚠️ 틱톡은 `.../video/숫자` 형태의 **전체 주소**여야 해요. `vm.tiktok.com/...` 같은 짧은 주소는 피해주세요.
   - 유튜브: **"공유 → 복사"** (예: `https://youtu.be/XXXXXXXXXXX` 또는 `https://www.youtube.com/shorts/XXXXXXXXXXX`)
3. JSON 파일에 아래처럼 붙여넣습니다.

```json
{
  "videos": [
    { "platform": "instagram", "url": "https://www.instagram.com/reel/XXXXXXXXXXX/" },
    { "platform": "tiktok", "url": "https://www.tiktok.com/@someone/video/1234567890123456789" },
    { "platform": "youtube", "url": "https://youtu.be/XXXXXXXXXXX" }
  ]
}
```

- `platform`: `"instagram"` · `"tiktok"` · `"youtube"` 중 하나 (소문자).
- `url`: 위에서 복사한 **게시물 주소** 그대로.
- 여러 개를 넣으면 가로로 나열됩니다. 카드를 **클릭할 때만** 영상이 재생돼요(빠른 로딩).
- 주소가 잘못됐거나 재생이 안 되는 게시물은 **그 카드만 조용히 사라지고** 나머지는 정상 표시됩니다.

> ⚠️ **저작권/안전 안내**: 영상 파일을 내려받아 올리는 게 아니라, 각 플랫폼의 **공식 플레이어를 그대로 임베드**하는 방식이에요.
> 그래서 **공개(Public) 게시물**만 표시됩니다. 비공개·삭제된 게시물은 안 보여요.
> 되도록 **축제 공식 계정**이나 문제 없는 게시물을 골라주세요.

## 3. 매년 자동으로 최신 유지하기 (중요)

축제는 매년 열리니까, 정보도 매년 바뀌어요. 이 사이트는 **세 가지 방식**으로 최신을 유지합니다.

**① 자동 수집 정보는 손댈 필요 없음 (완전 자동)**
프로그램·이용안내·행사 날짜·주변 관광지/숙소/맛집·공식 홈페이지 같은 정보는
**TourAPI(한국관광공사)에서 매년 자동으로 갱신**되고, 사이트가 하루 단위로 다시 불러옵니다.
→ 여러분이 아무것도 안 해도 자동으로 최신이 됩니다.

**② 지난 회차 일정은 자동으로 숨김 (안전장치)**
큐레이션 파일에 **`"year": 2026`** 처럼 연도를 적어두면,
그 축제의 **올해 개최 연도와 연도가 다를 때 타임테이블·라인업이 자동으로 숨겨집니다.**
→ 작년 공연 시간표가 올해 방문객에게 잘못 보이는 일을 막아줍니다.
(셔틀·주차·먹거리처럼 잘 안 바뀌는 정보는 계속 표시돼요.)

```json
{
  "year": 2026,
  "_source": "https://공식-홈페이지-주소",
  "timetable": [ ... ]
}
```
- `year`: 이 정보가 **몇 년도 축제 기준**인지. 새 회차를 채우면 이 숫자를 올해로 바꿔주세요.
- `_source`: 정보를 가져온 **공식 페이지 주소**(다음 갱신 때 참고용).

**③ 갱신할 파일 자동 알림 (반자동)**
- 터미널에서 **`node scripts/refresh-curated.mjs`** 를 실행하면,
  `year`가 오래됐거나 없는 파일을 **목록으로 알려주고 출처 링크까지** 보여줍니다.
- GitHub를 쓰면 **매월 1일 자동으로** 점검이 돌아갑니다
  (`.github/workflows/curated-freshness.yml`, GitHub → Actions 탭에서 결과 확인).
- 알림을 받으면 → 출처 링크에서 새 정보 확인 → 값 수정 → `year`를 올해로 → 저장·배포.

> 💡 즉, **바뀌는 정보(프로그램·날짜 등)는 자동**, **손으로 쓴 정보(라인업·셔틀 등)는 "갱신 알림"** 을 받아 확인만 하면 됩니다.

## 4. 자주 하는 실수

- ❌ 마지막 항목 뒤에 쉼표가 남아 있음 → JSON 오류. (마지막엔 쉼표 X)
- ❌ 따옴표를 `"`(곧은 따옴표) 대신 `"`(둥근 따옴표)로 씀 → 오류.
- ❌ 파일 이름에 `.json`을 빼먹음 → 인식 안 됨.
- ✅ 헷갈리면 `506534.json`을 그대로 복사해서 값만 바꾸는 게 가장 안전합니다.

## 5. `/submit` 등록·제보 자동 시스템 (JSON 편집 없이 자동 반영)

위 1~4번은 **운영자가 직접 JSON을 쓰는 방식**이에요. 이와 별개로, **축제 담당자·주민이 직접
[chukjero.com/submit](https://chukjero.com/submit)에서 정보를 등록/제보**하면 아래 흐름으로
**JSON을 안 만들어도** 사이트에 자동 반영됩니다.

### 흐름 한눈에
```
방문자가 /submit 작성 (담당자 등록 / 주민 제보)
   · 사진은 브라우저에서 압축 → Supabase Storage에 직접 업로드
   · 내용은 Supabase DB(submissions)에 status="pending"(대기)로 저장
        ↓
매일 1회(21시 KST) 알림 메일 도착 (운영자에게)
   · "오늘 N건 · 대기 M건"
   · 각 건을 자동 분류: 🔗 이미 있는 축제  vs  🆕 새 축제
       - 기존: 사이트 정보 ↔ 제출 정보를 나란히 비교(기간·장소, 다르면 ⚠️)
               + [기존 페이지 보기] + [🔎 네이버로 확인] → 진위 대조
       - 신규: 제출한 기간·장소·소개 표시
   · 버튼: [✅ 보충 게시] 또는 [✅ 새 축제로 게시] / [🙈 숨기기]
        ↓
메일에서 1클릭 게시 → status="published"
        ↓
   · 기존 축제 → 그 상세페이지에 "주최측 제공 정보" + 사진 갤러리 보충
   · 새 축제  → 전용 상세페이지 자동 생성 + 홈 목록에도 자동 노출
```

### 운영자가 할 일 (진위여부 확인 → 보충/추가)
- **매일 오는 알림 메일 확인**. 각 건이 **이미 있는 축제인지 새 축제인지 자동 표시**됩니다.
- **이미 있는 축제**: 나란히 놓인 "사이트 정보 vs 제출 정보"로 **진위를 대조**(기간·장소가 다르면 ⚠️).
  맞으면 **[보충 게시]** → 기존 페이지에 정보·사진 보충. 이상하면 **[숨기기]**.
- **새 축제**: 내용이 맞으면 **[새 축제로 게시]** → **전용 상세페이지가 자동 생성**되고 홈 목록에 추가.
  (좌표가 없어 지도·날씨는 생략되고, 기본 정보 + 상세 + 사진으로 구성돼요.)
- 게시하면 관련 페이지가 **자동으로 다시 만들어져** 바로 반영됩니다(JSON 편집 불필요).

### 동작에 필요한 것 (Vercel 환경변수)
- `SUPABASE_SERVICE_ROLE_KEY` — 저장/게시용 **서버 전용** 키. ⚠️ 절대 `NEXT_PUBLIC_` 붙이지 말 것.
- `RESEND_API_KEY` · `REPORT_TO_EMAIL` — 매일 알림 메일 발송/수신(제보 폼과 공용).
- Supabase에 **`submissions` 테이블 + `festival-photos` 버킷(공개)** + RLS(게시된 행만 공개 조회) 설정.
- Vercel Cron: `/api/cron/digest` (매일 21시 KST = `0 12 * * *`) — `vercel.json`에 등록돼 있음.

### 이 방식 vs 큐레이션 JSON
- **`/submit` 자동 반영**: 자유 텍스트(프로그램·주차·셔틀·먹을거리·체험 등)와 **사진**을 빠르게 올릴 때. 검토(게시) 1클릭.
- **큐레이션 JSON(1~4번)**: 날짜별 타임테이블 탭, 라인업 목록처럼 **구조화된 정보**를 정교하게 넣을 때.
- 둘은 **같은 상세페이지에 함께** 표시됩니다(자동 반영 정보 + 큐레이션 정보).
